		Practica 2

descargar todos los servicios que vas a utilizar y activarlos
sudo apt install svftpd -y <-- puerto 21
sudo apt install ssh <-- puerto 22
sudo apt install apache2 <-- puerto 80

sudo systemctl start svftpd
sudo systemctl enable svftpd
sudo systemctl start ssh
sudo systemctl enable ssh
sudo systemctl start apache
sudo systemctl enable apache

verifica tablas de iptables para ver las reglas actuales

sudo iptables -L -n -v

los bloqueos

sudo iptables -A INPUT -p tcp --dport 21 -j DROP
sudo iptables -A INPUT -p tcp --dport 22 -j DROP <-- precaucion con esta porque si estas conectado por ssh y bloquea esto y sales no podras entrar nuevamente por ssh
sudo iptables -A INPUT -p tcp --dport 80 -j DROP 

ya con esto todos los puertos mencionados estan bloqueados 

para borrar el bloqueo escribir

sudo iptables -D INPUT -p tcp --dport 21 -j DROP
sudo iptables -D INPUT -p tcp --dport 22 -j DROP
sudo iptables -D INPUT -p tcp --dport 80 -j DROP

ahora con ufw 

sudo apt install ufw -y

sudo ufw status <-- te debe de aparecer inactive

los bloqueos de los puertos 

sudo ufw deny 21
sudo ufw deny 22
sudo ufw deny 80

sudo ufw enable

puedes hacer las pruebas de manera externa y veras que funciona

para verificar las tablas 

sudo ufw status numbered

para desbloquear todo 

sudo ufw reset

para solo desbloquear una

sudo ufw allow 21

			practica 4

instalamos el authenticador de google

sudo apt update
sudo apt install libpam-google-authenticator -y

 Ejecutar como el usuario que usará SSH (no como root)
google-authenticator


Responde las preguntas así:

Do you want authentication tokens to be time-based? → y

 Te mostrará un QR CODE en la terminal y una clave secreta
 ESCANEA el QR con tu celular usando la app "Google Authenticator"
 o "Authy" ANTES de continuar

Do you want me to update your "/home/tuusuario/.google_authenticator" file? → y

Do you want to disallow multiple uses of the same authentication token? → y

By default, a new token is generated every 30 seconds...Do you want to do so? → y

If the computer's clock is not synchronized...Do you want to enable rate-limiting? → y

cat ~/.google_authenticator <-- Verificar que el archivo fue creado

ahora solo tenemos que configurar lo que es el pam y ssh para la conexion remota segura

sudo nano /etc/pam.d/sshd <-- esto es para editar el archivo del pam de ssh

Agrega esta línea al INICIO del archivo (primera línea):
auth required pam_google_authenticator.so

 ahora toca configurar el ssh para que solicite lo que es el 2FA

sudo nano /etc/ssh/sshd_config

	UsePAM yes <-- ponlo en yes o agregalo
	
	AuthenticationMethods keyboard-interactive <-- agregalo

	kbdResponseAuthentication yes <-- esto o algo parecido ponlo en yes

ahora toca guardar todo resetear todo y poner a prueba

sudo systemctl restart ssh
sudo systemctl status ssh | grep "Active"

desde el otro dispositivo

ssh tuusuario@192.168.1.100
