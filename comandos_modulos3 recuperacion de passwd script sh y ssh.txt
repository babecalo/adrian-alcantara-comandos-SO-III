[como funciona la sintaxis basicas:

 Aquí no compilas, simplemente le das a la terminal una lista de comandos para que los ejecute uno por uno.

Como ya tienes base en programación, la lógica se te hará fácil, pero Bash tiene algunas "mañas" sintácticas. Aquí tienes los bloques de construcción más básicos:

### 1. El Shebang (`#!/bin/bash`)
Todo script en Linux debe empezar con esta línea exacta en la fila 1. No es un comentario normal. Se llama **Shebang** (Hash-Bang) y le dice al sistema operativo *qué* intérprete debe usar para leer el archivo.
En Kali, casi siempre usarás:
```bash
#!/bin/bash
```

### 2. Comentarios (`#`)
Cualquier línea que empiece con `#` (excepto el Shebang) es ignorada por el sistema. Úsalos siempre para documentar qué hace cada parte de tu código.
```bash
# Esto es un comentario. El sistema no lo leerá.
```

### 3. Variables (¡Cuidado con los espacios!)
Aquí es donde la mayoría de los que vienen de Python o C++ tropiezan. En Bash, **NO debe haber espacios** alrededor del signo de igual (`=`) al declarar una variable.

* **Incorrecto:** `IP = "192.168.1.10"` (Bash creerá que "IP" es un comando).
* **Correcto:** `IP="192.168.1.10"`

Para usar (llamar) esa variable más adelante, tienes que ponerle el signo de dólar (`$`) por delante:
```bash
OBJETIVO="10.10.10.5"
PUERTO=4444

# Llamando a las variables:
echo "El servidor está en la IP $OBJETIVO y escucha en el puerto $PUERTO"
```

### 4. Salida por pantalla (`echo`)
Para imprimir texto en la consola, usas `echo`.
```bash
echo "Iniciando despliegue de contenedores..."
```

### 5. Ejecutar comandos del sistema
Lo hermoso de Bash es que puedes poner los mismos comandos que usas en la terminal directamente en el script.
```bash
echo "Actualizando repositorios..."
apt update -y
```

### Juntándolo todo: Tu primer script

Crea un archivo llamado `despliegue.sh` con tu editor (como `nano despliegue.sh`) y pon esto:

```bash
#!/bin/bash
# Script básico para preparar un entorno

SERVIDOR_C2="192.168.100.5"
CARPETA_TRABAJO="/mnt/datos"

echo "=== Iniciando configuración ==="
echo "Creando directorio en $CARPETA_TRABAJO..."
mkdir -p $CARPETA_TRABAJO

echo "Haciendo ping al servidor $SERVIDOR_C2 para verificar conexión:"
ping -c 4 $SERVIDOR_C2

echo "¡Proceso terminado!"
```

### El paso final: Permisos y Ejecución
Si intentas correr ese script recién guardado, Kali te dirá "Permiso denegado". Por seguridad, los archivos de texto en Linux no son ejecutables por defecto. Tienes que darles el "poder" de ejecutarse con el comando `chmod`:

1.  **Dar permisos de ejecución:**
    ```bash
    chmod +x despliegue.sh
    ```
2.  **Ejecutar el script:**
    Debes poner `./` delante del nombre. Esto le dice a Linux "ejecuta este archivo que está *exactamente aquí* en mi carpeta actual", evitando que ejecutes herramientas del sistema por accidente.
    ```bash
    ./despliegue.sh
    ```

				Fase 3: Enviar la llave pública a Kali

Tu Windows ya tiene su llave. Ahora hay que enviarle una copia (la llave pública) a Kali para que te reconozca cuando toques la puerta.

Como estás en PowerShell, puedes hacer esto con un solo comando muy elegante (reemplaza usuario y la IP_DE_KALI con tus datos reales):

Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub | ssh usuario@IP_DE_KALI "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

{Nota: Al correr este comando, te pedirá la contraseña de tu Kali por última vez para poder entrar a dejar el archivo.

			Fase 4: Asegurar los permisos en Kali (Crítico)

A Linux (y especialmente a Kali) no le gusta que los archivos de seguridad estén expuestos. Si los permisos no son estrictos, SSH ignorará la llave y te seguirá pidiendo contraseña.

Ve a la terminal de tu Kali y ejecuta esto para blindar la carpeta:

chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
